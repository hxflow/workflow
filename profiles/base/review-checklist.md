# 通用代码审查清单（Base Review Checklist）

> Phase 05 使用，逐项检查 diff 中的变更。
> 团队专属检查项在各自 review-checklist.md 中补充。

## 必须修复（阻断合并）

- [ ] **架构层级合规** — 是否存在跨层违规导入（内层不可导入外层）
- [ ] **GP-BASE-003** — 无裸抛原生异常，统一使用错误类型
- [ ] **GP-BASE-001** — 源码目录中无调试日志（console.log / fmt.Println / print 等）
- [ ] **GP-BASE-007** — 无魔法数字，配置值集中管理
- [ ] **GP-BASE-005** — 无不安全的类型断言

## 建议修复

- [ ] **GP-BASE-009** — 核心业务方法是否有对应单元测试
- [ ] **GP-BASE-004** — 业务层是否有过度防御性捕获
- [ ] **GP-BASE-002** — 结构化日志字段是否完整
- [ ] **文档同步** — 接口变更是否同步到需求文档

## 观察项

- [ ] 新增外部依赖是否合理
- [ ] 是否存在 N+1 查询或性能隐患
- [ ] 命名规范一致性
