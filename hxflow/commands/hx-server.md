# 启动项目服务

## 执行步骤

1. 执行 `hx-server [<target>]`，读取已记录的服务启动方式。
2. 若返回已记录服务，用后台 shell 在其 `cwd` 执行 `command`，观察日志直到出现监听地址或明确失败。
3. 若记录缺失或启动失败，分析项目结构、README、脚本、入口文件和环境示例，继续尝试直到找到可成功启动的命令。
4. 启动成功后执行 `hx-server save <id> --cwd <cwd> --command <command> --type <type> [--url <url>]` 写回记录。
5. 输出服务类型、执行目录、命令和访问地址；无法启动时说明具体阻断。

## 约束

- 只启动项目已有服务，不修改配置、不修业务代码
- 前端服务以可打开页面为成功，后端服务以可监听端口或健康接口响应为成功
- 缺依赖、端口冲突或环境变量缺失时，说明阻断并停止
- 不把 server 当作 review、test 或端到端验收
- save 只记录已经真实启动成功的命令
