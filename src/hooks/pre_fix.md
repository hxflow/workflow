# pre_fix

在 `hx-fix` 执行前，优先补齐缺陷详情。

## 触发条件

- 当前输入或当前会话上下文中出现企业内部 DevOps 系统缺陷引用，例如无双缺陷

## 执行步骤

1. 在当前输入和会话上下文中查找企业内部 DevOps 系统缺陷引用
2. 若未找到缺陷引用，直接跳过，不阻断 `hx-fix`
3. 若找到缺陷引用，获取缺陷详情
4. 将缺陷详情整理为后续可直接消费的修复上下文，至少补齐：
   - 缺陷标题
   - 缺陷描述
   - 复现信息
   - 来源引用
5. 后续 `hx-fix` 直接基于已补齐的修复上下文定位并修复问题，不再重复询问用户基础缺陷信息

## 约束

- 只负责“拉取并整理缺陷详情”，不直接修改代码
- 外部查询失败时，明确说明失败原因，不静默编造缺陷
- 不重算、不覆盖已确定的 `feature`
- 若缺陷详情不足以支撑修复，应显式指出缺失项
- 不写死具体实现方式
