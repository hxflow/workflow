# pre_doc

在 `hx-doc` 执行前，优先补齐需求详情。

## 触发条件

- 当前输入或当前会话上下文中出现企业内部 DevOps 系统任务引用，例如无双任务

## 执行步骤

1. 在当前输入和会话上下文中查找企业内部 DevOps 系统任务引用
2. 若未找到任务引用，直接跳过，不阻断 `hx-doc`
3. 若找到任务引用，获取需求详情
4. 将任务详情整理为后续可直接消费的需求上下文，至少补齐：
   - 需求标题
   - 需求描述
   - 验收要点
   - 来源引用
5. 后续 `hx-doc` 直接基于已补齐的需求上下文生成需求文档，不再重复询问用户基础需求信息

## 约束

- 只负责“拉取并整理需求详情”，不直接创建或修改需求文档
- 外部查询失败时，明确说明失败原因，不静默编造需求
- 不重算、不覆盖已确定的 `feature`
- 若任务详情不足以支撑需求文档生成，应显式指出缺失项
- 不写死具体实现方式
