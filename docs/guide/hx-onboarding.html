First Look

命名与职责

区分产品名、命令名和包名。

概念层

产品名与命令名

HXFlow 是产品名，终端命令为 hx，npm 包名为 @hxflow/cli。

使用层

hx 的职责

负责组织文档、计划、Profile 规则与上下文，并生成执行 Prompt 或运行校验命令。

i
顺序：执行链路 → 运行机制 → 配置参考。
Install

安装

安装 npm 包后执行 hx setup 初始化全局资源，然后在 Claude Code 中运行 /hx-init 初始化项目。

终端操作

安装与验证

# 全局安装 CLI
npm install -g @hxflow/cli --registry https://npm.cdfsunrise.com/

# 查看版本
hx version

# 初始化全局资源（仅需运行一次）
hx setup
资源写入

hx setup 写入目录

~/.hx/config.yaml
~/.hx/commands/
~/.hx/profiles/
~/.hx/pipelines/
~/.claude/commands/  ← 转发器

转发器不含命令逻辑，运行时按三层优先级路由到实体命令文件；skill 由 Agent 自身机制管理。

Init

项目初始化（Claude 命令）

在 Claude Code 中运行 /hx-init；可选 --profile 指定要使用的 Profile 名称。框架默认内置 base。

# 在 Claude Code 中运行（自动分析项目结构）
/hx-init

# 或显式指定 profile
/hx-init --profile base
输出

写入的文件

.hx/config.yaml       ← defaultProfile + paths
CLAUDE.md             ← 注入 harness 标记块
.hx/profiles/<name>/  ← 自定义 profile（如需）
约束

项目主入口是 .hx/config.yaml

Claude 自动分析技术栈、门控命令和目录结构，推断 paths 配置。与默认值相同的 paths 字段不写入，保持简洁。

Workflow

执行链路

支持一键自动化（/hx-go）和手动逐步执行两种模式。

✓
推荐：使用 /hx-go user-login --profile base 一键执行全流程，4 个检查点等待人工确认。这里的 user-login 就是系统内部的 feature key。
01
创建需求文档
执行 /hx-doc --profile base 创建需求文档；可用 --task <id> 直接从任务系统拉取需求来源。feature key 由系统自动生成，正常情况下不用关心。
02
生成执行计划
补全验收标准后执行 /hx-plan --profile base；系统会自动从当前需求文档续接并生成计划 Markdown 和 *-progress.json。
03
执行需求
执行 /hx-run --profile base，按计划顺序完成当前需求下的全部 pending 任务；命令会先自动完成当前需求的执行前预检。仅在调试/恢复时才传 --task。
04
门控与收口
执行 /hx-qa --profile base 运行质量校验。正常情况下进度已由 /hx-run 自动回写，无需额外手动标记任务完成状态。
# 手动逐步执行
/hx-doc --task 12345 --profile base
/hx-plan --profile base
/hx-run --profile base
/hx-qa --profile base

# 或一键执行
/hx-go user-login --profile base

# 从指定步骤恢复
/hx-go user-login --from run --profile base
✓
可选诊断：若 /hx-run 因输入不完整而停止，可额外执行 /hx-ctx --profile base 查看具体缺失项。
Pitfalls

常见问题

以下错误会直接影响执行结果或上下文一致性。

错误 01

执行了 hx plan，却没先补需求文档

计划直接依赖当前需求文档内容。不完整的 AC 会被固化到计划和进度文件。

错误 02

误将 /hx-run 视为代码执行器

/hx-run 是 Claude 命令，由 Claude Code 读取并执行，不是 Shell 命令行工具。

错误 03

没有先执行 hx setup，或没有在 Claude Code 中运行 /hx-init

hx setup 初始化全局转发器；/hx-init 初始化项目配置。两步都必须完成。

错误 04

将 hx setup 和 /hx-init 搞混

hx setup 只需运行一次（机器级），/hx-init 在每个新项目运行（项目级）。

错误 05

任务做完了忘记 hx done

这会导致 *-progress.json 与实际状态不一致，并影响后续执行与交付。
